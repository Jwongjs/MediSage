SP1 Design — Section 1: Architecture Before vs After

What the inference layer looks like before and after SP1

BEFORE — Local Model Stack
main.py startup
→ model_manager.load_all_models()
→ loads 4GB GGUF into RAM/VRAM
→ loads EfficientNetB0
→ loads sentence-transformers
Nodes
LLMDiagnosisNode → LocalLlamaAdapter
FollowUpNode → LocalLlamaAdapter
OverallAnalysisNode → LocalLlamaAdapter
ImageClassificationNode → EfficientNet
MedicalReportNode → LocalLlamaAdapter
Infrastructure
websocket_manager.py (unused)
ConnectionManager (commented out)
CUDA dependency in Docker image
~6GB Docker image
Cannot scale horizontally
Docker image requires GPU
6GB+ image size
AFTER — API Inference Stack
main.py startup
→ validate env vars (LLM_API_KEY etc.)
→ test LLM connectivity (health ping)
→ no model loading
Nodes
LLMDiagnosisNode → LLMClient
FollowUpNode → LLMClient
OverallAnalysisNode → LLMClient
ImageClassificationNode → REMOVED
MedicalReportNode → LLMClient
New: backend/llm/client.py
OpenAI-compatible async client
base_url + model + api_key from env
Now: Groq · Future: vLLM on Modal
Stateless — scales horizontally
Tiny Docker image (~200MB)
Swap provider via env vars only
